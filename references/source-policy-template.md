## Source classes

| class | examples | allowed? | citation rule | notes |
|---|---|---:|---|---|
| Primary official | specifications, vendor docs, standards, statutes, official repos | yes | cite directly | preferred |
| Primary data | datasets, API responses, schema exports | yes | cite source and access date when useful | preserve resource URI |
| Secondary expert | reputable analysis, academic papers, recognized industry research | yes | cite for interpretation | avoid substituting for primary facts |
| News | reputable reporting | conditional | cite dates and event claims | use for chronology |
| Community | forums, issue comments, Stack Overflow, Reddit | conditional | cite as anecdotal/community evidence | do not generalize without support |
| AI-generated pages | generated summaries without clear sources | no | do not cite | use only to discover leads |

## Citation bar

- Every non-obvious external claim needs a source.
- Recent, disputed, legal, medical, financial, or security claims need stronger sourcing.
- If a fact cannot be sourced, mark it under `# Open questions` instead of asserting it.

## Licensing and reuse

- Do not paste long copyrighted text.
- Summarize and link.
- Track source licenses if the bundle will be published.
