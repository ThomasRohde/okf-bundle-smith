# Summary

This example bundle is sourced from the upstream
[OKF specification](/sources/okf-spec.md) and local OKF Bundle Smith
documentation. It is not a market landscape, product comparison, or broad web
research bundle.

# Authoritative Sources

The canonical public source for format requirements is the
[OKF specification source](/sources/okf-spec.md). Local source identifiers
include `references/okf-v0.1-cheatsheet.md`,
`references/okf-v0.1-conformance.md`, `references/concept-type-catalog.md`,
`README.md`, and `skills/okf-bundle-architect/SKILL.md`.

# Citation Rules

Claims about hard OKF requirements should cite the public specification. Claims
about OKF Bundle Smith behavior should cite local repository files by path in
the `# Citations` section. Local paths are cited as source identifiers instead
of Markdown links so the bundle remains valid when linted from its own root.

# Exclusions

This bundle does not use secondary blog posts, vendor marketing pages, or
uncited model knowledge. It also does not invent canonical URIs for concepts
that do not have one.

# Relationships

* [Open Knowledge Format](/concepts/open-knowledge-format.md) relies on this source boundary.
* The [validation profile](/controls/validation-profile.md) enforces citation and link quality where tooling can detect it.
