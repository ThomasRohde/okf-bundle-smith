# Summary

The OKF specification source is the canonical public reference for the minimal
[Open Knowledge Format](/concepts/open-knowledge-format.md) rules used in this
bundle. It defines the core contract for Markdown concept files, YAML
frontmatter, and required metadata.

# Reliability

This source is treated as authoritative for OKF v0.1 conformance. Local OKF
Bundle Smith references interpret that source for authoring and validation, but
they do not replace the upstream specification.

# Used For

* Defining the required [frontmatter](/concepts/frontmatter-schema.md) field.
* Explaining [concept files](/concepts/concept-file.md).
* Distinguishing reserved navigation files from concept files.
